This is a relaunch of my blog.  I haven't posted anything since 2016 and
whatever I posted so long ago is long forgotten.

My last attempt used HTML and CSS only, and that proved to be quite 
difficult.  This time around, I'm going to use Jekyll, mainly because it
comes with GitHub by default.  Jekyll is a static website generator that 
specializes in blogging. Users can use a visual editor to create content, 
the platform will then transform it into a static page that is ready to 
be viewed. This has multiple advantages.  For example, a static page 
loads much faster than a dynamic page. Another would be that since the 
website is static, it has no database. That means there is nothing for 
a hacker to hack. However, there are also setbacks. For instance, the 
platform lacks CMS functionality, which means I am going to need to 
be comfortable using markdown to edit pages. I can do that on the GitHub 
app, though the GitHub website, or through a local git repository.  Free 
hosting on GitHub is also attractive to a hobbyist like me.

I did think of WordPress, which is an open-source CMS, that began as a 
blogging platform, and has evolved to do just about anything. This is 
all thanks to the high level of flexibility and customization users 
have at their disposal.  This is possible because of the huge library 
of plugins and themes the platform offers. Plugins add specific features
to a website - kind of like apps on a smartphone.  Themes determine the 
appearance of the website but can be customized regardless of your 
choice. With thousands of each to choose from, the only limitation in 
WordPress is my own own creativity.  It comes with an editor -
Gutenberg - but that is browser based.  The smartphone integration
that comes with the Github app, while a more limited editing solution, 
is just suits me better.  Wordpress also relies on a backend database 
and SQL injection vulnerabilities scare me a bit.  That's not even to
mention vulnerabilities in plugins. 

Having opted for Jekyll, step number one was to choose a theme.  First
I tried the "Gold Standard" Minimal Mistakes.  This was more difficult 
than HTML and CSS.  Minima was suggested to me as a good alternative.  It
is smaller, but still has lots of configuration options to learn.  If I
have to climb a learning curve, then it might as well be the learning 
curve of Jekyll itself, so I've created a really minimal theme for myself.
Maybe I'll regret that and go back to Minima later, but right now the
focus is on content rather than presentation.  Having said that, I think
I've done a passable job on the presentation too.
